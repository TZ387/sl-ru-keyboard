# sl-ru-keyboard

Russian Cyrillic keyboard layout for Slovenian (and Croatian/Serbian Latin) keyboards on Ubuntu 24.

This layout maps Russian (and other Cyrillic) characters to the **physically equivalent keys on a Slovenian keyboard** вҖ” so the key where you'd type `ДҢ` produces `Р§`, `Е ` produces `РЁ`, and so on. The AltGr layer extends the layout to cover characters from **6 Cyrillic-script languages**: Russian, Belarusian, Bulgarian, Macedonian, Serbian Cyrillic, and Ukrainian.

This is an updated and working version of [anzeljg/ruska_sl](https://github.com/anzeljg/ruska_sl), adapted for **Ubuntu 24.04** (the original instructions were outdated вҖ” they referenced files and steps that no longer work on modern Ubuntu).

**Key levels:**
- Plain key вҶ’ base character (e.g. `Q` вҶ’ `СӢ`)
- `Shift` + key вҶ’ uppercase (e.g. `Shift+Q` вҶ’ `Р«`)
- `AltGr` + key вҶ’ extended Cyrillic character from another language (e.g. `AltGr+Q` вҶ’ `С—`)
- `AltGr+Shift` + key вҶ’ uppercase of extended character (e.g. `AltGr+Shift+Q` вҶ’ `РҮ`)

## Installation on Ubuntu 24.04

> **Note:** The installation method for Ubuntu 24 differs from older guides. The key differences are:
> - You must edit `evdev.xml` (not `base.xml` or `base.lst` as older guides say)
> - You need to clear the XKB cache after changes
> - A full logout/login or reboot is required for GNOME to pick up the new layout

### Step 1 вҖ” Copy the symbol file

```bash
sudo cp ru_sl /usr/share/X11/xkb/symbols/ru_sl
```

### Step 2 вҖ” Register the layout in evdev.xml

Open the file:

```bash
sudo nano /usr/share/X11/xkb/rules/evdev.xml
```

Find the `<layoutList>` section and add the following entry (you can place it near the other `ru` entries):

```xml
<layout>
  <configItem>
    <name>ru_sl</name>
    <shortDescription>ru_sl</shortDescription>
    <description>Russian (Slovenian keyboard)</description>
    <languageList>
      <iso639Id>rus</iso639Id>
    </languageList>
  </configItem>
</layout>
```

Save and exit (`Ctrl+O`, `Enter`, `Ctrl+X`).

### Step 3 вҖ” Clear the XKB cache

Ubuntu caches compiled keyboard layouts. You must clear this cache or your changes will be ignored:

```bash
sudo rm -rf /var/lib/xkb/*
```

### Step 4 вҖ” Log out and log back in

A full logout/login (or reboot) is required for GNOME Settings to discover the new layout.

### Step 5 вҖ” Add the layout in GNOME Settings

1. Open **Settings вҶ’ Keyboard вҶ’ Input Sources**
2. Click **+**
3. Select **Other** (or search for "Russian")
4. Find **Russian (Slovenian keyboard)** and add it

To switch between layouts, you can set a shortcut under **Settings вҶ’ Keyboard вҶ’ Keyboard Shortcuts вҶ’ Switch input source** (e.g. `Super+Space` or `LShift+LCtrl`).

**Note**: For switching the language layout using `Super+Space`, you have to press and hold the `Super` key and then click `Space` as many times as required.

### Optional: Test without logging out

You can test the layout immediately in the current session (this does not affect GNOME Settings):

```bash
setxkbmap ru_sl
```

To revert to the Slovenian layout:

```bash
setxkbmap si
```

## Applicability to Croatian and Serbian Latin keyboards

The Slovenian, Croatian, and Serbian Latin keyboard layouts share the same physical key positions for the special characters (`ДҢ`, `Е `, `ЕҪ`, `Дҗ`, `ДҶ`). This means this layout works equally well if your physical keyboard is labeled as Croatian or Serbian Latin вҖ” the Cyrillic character mappings are identical.

The layout is designed around the **positional logic** of the QWERTZ South Slavic keyboards, not Slovenian-specific keys, so the mapping feels natural on all three variants.

## How it works вҖ” why the AltGr layer covers so many languages

The XKB symbol file defines up to 4 levels per key:
1. Base (plain keypress)
2. Shift
3. AltGr
4. AltGr+Shift

The base and Shift levels cover standard Russian Cyrillic. The AltGr and AltGr+Shift levels were used to place characters from other Cyrillic-script languages that don't exist in Russian вҖ” Ukrainian `С–/С—/С”/Т‘`, Belarusian `Сһ`, Macedonian `С•/С“/Сң`, and Serbian `С’/Сӣ/Сҳ/Сҷ/Сҡ/Сҹ`. This means a single layout file gives you access to all six languages without switching layouts.
